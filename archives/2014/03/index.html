Node.js学习之'一个简单的nodejs web应用'

很早就听说过nodejs的威力了，一直想学习，但是一直比较忙，而且自己也是举棋不定，初出茅庐的我，对众多技术比较迷茫，什么都想学，最后什么的没开始学。这次我是下定决心一定好静下心来慢慢学，就从nodejs开始吧。网上找到一份77页的nodejs入门手册，我几乎是花了两个晚上就看完了，然后跟着书本搭建了web服务，可以说nodejs的入门还是比较简单的，只要有点javascript的知识都能上手。下面我就把根据手册一步步实现的一个web小应用的步骤记录下来，就当是温故吧！
首先介绍一下nodejs。nodejs是一个javascript框架，是一个让javascript运行种在服务器段的开发平台。nodejs内建了HTTP服务的支持，也就是说你不用在自己去搭建一个Apache之类的服务器了，你可以使用nodejs轻松实现一个网站和服务器的组合。nodejs最大的特点是采用异步式I/O与事件驱动的架构设计。对于高并发的解决方案，传统的架构是多线程模型，而nodejs是单线程模型，对于所有I/O请求采用异步式的请求方式。nodejs开发是以模块的方式进行的，把各个功能拆分、组装，通过require来调用其他模块。总结自BYvoid大神的《Node.js开发指南》 简单了解后我们开始吧。
首先，既然是一个web应用，就必然需要一个应用服务器来处理应用产生的各种HTTP请求，前面提到nodejs内建了HTTP服务，我们可以用很简单的代码来建立一个服务器。代码如下server.js:

var http = require("http"); 
var url = require("url");
function start(route, handle) { 
function onRequest(request, response) {
 var pathname = url.parse(request.url).pathname; 
 console.log("Request for " + pathname + " received."); 
 route(handle, pathname, response, request); 
 } 
 http.createServer(onRequest).listen(8888);
 console.log("Server has started.");
 }
 exports.start = start;

第一行代码就是调用了nodejs的内建HTTP服务。http.createServer这个函数会返回一个对象，这个对象有一个叫做 listen 的方法，这个方法有一个数值参数，指定这个HTTP服务器监听的端口号。最后一行代码表示这个模块通过名称start暴露出来。 我们看到onRequest方法里有一个route方法，很奇怪，哪里来的呢？route.js: function route(handle, pathname, response, request) { console.log("About to route a request for " + pathname); if (typeof handle[pathname] === 'function') { handlepathname; } else { console.log("No request handler found for " + pathname); response.writeHead(404, {"Content-Type": "text/html"}); response.write("404 Not found"); response.end(); } } exports.route = route;

上面这段代码的功能我们从文件名route就能猜到了。对的，就是请求路径的路由转发作用。handle暂且不管，pathname就是server.js中对请求解析好传过来的路径名称，在route中它会指定handle具体去进行什么样的操作。那么handle到底长什么样呢？

构建基于rest的Web服务

环境准备：

　　一种集成开发环境IDE;(本文使用Spring Tool Suite (STS) )
　　JDK 1.6及以上;
　　Gradle 1.8+ or Maven 3.0+(本文使用maven 3.1.1);

新建项目：

如果你对新建maven项目不是很熟悉，请参照spring官网创建maven项目的简单教程。

创建文件目录结构：

└── src └── main └── java └── hello

创建工具构建文件：

maven的pom.xml文件可以从这里获取。将这里的pom.xml文件直接覆盖你项目的pom.xml文件。

创建一个资源表示类

项目和环境打好之后，我们就可以创建我们的web服务了。
我们的web服务将处理一个名为/greeting，并任意的带有一个名为name参数的GET请求。GET请求将返回200 OK的响应，响应体内带有JSON格式数据，数据结构如下： 

{  
    "id": 1,  
    "content": "Hello, World!"  
}

在目录src/main/java/hello下新建文件Greeting.java。 

package hello;  
public class Greeting {  
    private final long id;
    private final String content;  
    public Greeting(long id, String content) {
    this.id = id;
    this.content = content;
}

   public long getId() {  
        return id;  
    }  

    public String getContent() {  
        return content;  
    }  
}

创建一个资源控制器

解析：用spring构建rest的web服务，HTTP请求用控制器来掌控。这个组件简单的被@Controller注解标识。下面的GreetingController为/greeting处理了GET方式的请求，并返回一个新的Greeting对象。@RequestMapping注解确保HTTP请求将/greeting映射到greeting()方法。@RequestParam将请求中的参数name绑定到方法中的name参数，请求中的参数不是必须的，如果没有的话，默认值World就会派上用场。传统的MVC控制器与基于rest的web服务的一个关键区别在于HTTP响应体的创建。与MVC在服务端呈现HTML格式的响应数据不同的是基于rest的web服务返回的是一个JSON对象。注解@ResponseBody的作用就是告诉spring MVC不要返回html而是返回一个对象。

在目录src/main/java/hello下新建文件GreetingController.java。 

package hello;

import java.util.concurrent.atomic.AtomicLong;
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.ResponseBody;

@Controller
public class GreetingController {

private static final String template = "Hello, %s!";
private final AtomicLong counter = new AtomicLong();

@RequestMapping("/greeting")
public @ResponseBody Greeting greeting(
        @RequestParam(value="name", required=false, defaultValue="World") String name) {
    return new Greeting(counter.incrementAndGet(),
                        String.format(template, name));
    }
}

使应用程序可执行

在目录src/main/java/hello下新建文件Application.java。
package hello;

import org.springframework.boot.autoconfigure.EnableAutoConfiguration;
import org.springframework.boot.SpringApplication;
import org.springframework.context.annotation.ComponentScan;

@ComponentScan
@EnableAutoConfiguration
public class Application {

    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

解析：注解@ComponentScan，告诉spring在包hello以及他的子类中递归的查找被直接或间接的标注了@Component的类。这确保spring能找到并注册标有@Controller的GreetingController，@Controller是@Component的一种

构建一个可执行JAR

如果你使用的是maven，你可以使用 mvn spring-boot:run来启动应用程序。或者你可以通过mvn clean package打包jar文件，然后用java -jar target/gs-rest-service-0.1.0.jar来运行jar文件。

运行服务

上面以及提到，如果你用的是maven，你可以直接用命令行mvn spring-boot:run.来启动应用程序。
